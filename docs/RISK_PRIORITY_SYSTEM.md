# 위치 기반 위험도 우선순위 시스템

## 개요

포트홀이 발생한 위치에 따라 위험도를 평가하고 우선순위를 부여하는 시스템입니다.
고속도로나 어린이 보호구역 등 사고 위험이 높은 지역의 포트홀을 우선적으로 처리할 수 있습니다.

## 위험도 가중치

### 위치 유형별 가중치

| 위치 유형 | 가중치 | 설명 |
|---------|--------|------|
| 고속도로 | 3.0 | 고속도로 및 자동차 전용도로 |
| 어린이 보호구역 | 3.0 | 어린이 보호구역 및 학교 주변 |
| 학교 주변 | 2.5 | 학교 주변 500m 이내 |
| 병원 주변 | 2.5 | 병원 및 응급의료기관 주변 |
| 시내버스 전용차로 | 2.0 | 시내버스 전용차로 |
| 주거 지역 | 1.2 | 주거 밀집 지역 |
| 상업 지역 | 1.0 | 상업 지역 |
| 일반 도로 | 1.0 | 일반 도로 |
| 공원/녹지 | 0.8 | 공원 및 녹지 지역 |

## 우선순위 점수 계산

우선순위 점수는 다음 공식으로 계산됩니다:

```
우선순위 점수 = (위치 가중치 × 10) + (깊이 점수) + (검증 점수)
```

- **위치 가중치**: 위치 유형에 따른 가중치 (주요 요소)
- **깊이 점수**: 깊이 비율 × 0.3 × 10 (0-3 범위)
- **검증 점수**: 검증 통과 시 1.0 × 0.2 × 10 = 2.0

### 위험도 레벨

우선순위 점수에 따라 위험도 레벨이 결정됩니다:

- **Critical (긴급)**: 우선순위 점수 ≥ 30
- **High (높음)**: 우선순위 점수 ≥ 20
- **Medium (보통)**: 우선순위 점수 ≥ 10
- **Low (낮음)**: 우선순위 점수 < 10

## 데이터베이스 스키마

### 추가된 필드

- `location_type`: 위치 유형 (highway, school_zone, etc.)
- `risk_level`: 위험도 레벨 (critical, high, medium, low)
- `priority_score`: 우선순위 점수 (DECIMAL)
- `location_description`: 위치 설명 (한글)

### 인덱스

- `idx_potholes_priority`: 우선순위 점수 인덱스 (내림차순)
- `idx_potholes_risk_level`: 위험도 레벨 인덱스
- `idx_potholes_location_type`: 위치 유형 인덱스

### 뷰

- `high_priority_potholes`: 우선순위 점수 2.0 이상의 포트홀
- `high_risk_potholes`: 위험도가 높거나 긴급한 포트홀

## 대시보드 기능

### 필터링

1. **우선순위 필터**
   - 최소 우선순위 점수 설정
   - 높은 우선순위 포트홀만 표시

2. **위험도 레벨 필터**
   - Critical, High, Medium, Low 선택
   - 다중 선택 가능

3. **위치 유형 필터**
   - 위치 유형별 필터링
   - 다중 선택 가능

### 시각화

1. **지도 마커**
   - 위험도에 따른 색상 및 크기
   - Critical: 진한 빨강, 크기 10
   - High: 빨강, 크기 8
   - Medium: 주황, 크기 6
   - Low: 노랑, 크기 5

2. **데이터 테이블**
   - 우선순위 점수 표시
   - 위험도 레벨 표시
   - 위치 설명 표시
   - 우선순위 점수 기준 정렬

## 사용 방법

### 1. 데이터베이스 마이그레이션

```sql
-- 마이그레이션 스크립트 실행
\i database/add_risk_fields.sql
```

또는 Docker에서:
```bash
docker-compose exec -T db psql -U pothole_user -d pothole_db < database/add_risk_fields.sql
```

### 2. AI Core 재시작

위험도 계산 로직이 적용되도록 AI Core를 재시작합니다:

```bash
docker-compose restart ai-core
```

### 3. 대시보드 확인

대시보드에서 다음을 확인할 수 있습니다:
- 우선순위 점수별 정렬
- 위험도 레벨별 색상 구분
- 필터링 기능

## 향후 개선 사항

1. **실제 위치 정보 연동**
   - 지도 API (Kakao Map, Naver Map) 연동
   - 실제 도로 유형 및 시설물 정보 확인
   - 정확한 위치 기반 위험도 판단

2. **동적 가중치 조정**
   - 시간대별 가중치 조정 (출퇴근 시간 등)
   - 날씨 정보 반영
   - 교통량 정보 반영

3. **알림 시스템**
   - Critical 레벨 포트홀 자동 알림
   - 우선순위 기반 작업 할당

4. **통계 및 분석**
   - 위치별 포트홀 발생 빈도 분석
   - 위험도 분포 분석
   - 우선순위 기반 처리 현황

## 참고

- 현재는 기본 위치 유형으로 설정되며, 실제 위치 정보는 향후 지도 API 연동 시 정확하게 판단됩니다.
- 우선순위 점수는 깊이와 검증 결과도 고려하므로, 위치가 같아도 깊이에 따라 우선순위가 달라질 수 있습니다.
