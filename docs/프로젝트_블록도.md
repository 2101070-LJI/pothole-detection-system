## 🔄 데이터 처리 흐름

```mermaid
sequenceDiagram
    participant User as 사용자
    participant Apache as Apache
    participant Flask as Flask App
    participant MySQL as MySQL DB
    participant Inference as Inference Container
    participant YOLO as YOLOv8
    participant NPU as NPU Worker
    participant Depth as Depth Model
    
    User->>Apache: HTTP 요청 (포트 80)
    Apache->>Flask: 요청 전달
    Flask->>MySQL: 데이터 조회
    MySQL-->>Flask: 포트홀 데이터
    Flask-->>Apache: JSON 응답
    Apache-->>User: 웹 페이지
    
    Note over Inference: 비디오 처리 시작
    Inference->>YOLO: 비디오 프레임 입력
    YOLO-->>Inference: 포트홀 탐지 결과
    Inference->>NPU: 깊이 검증 요청
    NPU->>Depth: 깊이 추정
    Depth-->>NPU: depth_ratio
    NPU-->>Inference: 검증 결과
    Inference->>MySQL: 검증 통과 데이터 저장
    Inference->>Flask: 처리 완료 알림
```

## 🔄 자동 파인튜닝 프로세스

## 🎯 핵심 포인트

1. **2개 컨테이너 구조**: LAMP + Inference
2. **Windows Host NPU Worker**: 깊이 검증 전용
3. **통합 챗봇**: LAMP 컨테이너 내부에서 실행
4. **자동화**: 파인튜닝 스케줄러 포함
5. **API 통합**: 카카오 Map API 연동
