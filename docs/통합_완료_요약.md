# 컨테이너 통합 완료 요약

## ✅ 통합 결과

**5개 컨테이너 → 2개 컨테이너 (60% 감소)**

### 통합 전
1. web-server (Apache)
2. dashboard (Streamlit)
3. ai-core (AI Engine)
4. db (PostgreSQL)
5. cloudflared (Cloudflare Tunnel)

### 통합 후
1. **app** (dashboard + ai-core 통합)
2. **db** (PostgreSQL)

## 📁 생성된 파일

- `app/Dockerfile` - 통합 컨테이너 빌드 파일
- `app/requirements.txt` - 통합 의존성
- `app/start.sh` - 시작 스크립트 (두 프로세스 실행)
- `app/.dockerignore` - 빌드 제외 파일
- `docker-compose-optimized.yml` - 최적화된 docker-compose 파일
- `컨테이너_통합_가이드.md` - 상세 가이드

## 🚀 사용 방법

```powershell
# 최적화된 버전 실행
docker-compose -f docker-compose-optimized.yml up -d

# 빌드 포함
docker-compose -f docker-compose-optimized.yml up -d --build

# 로그 확인
docker-compose -f docker-compose-optimized.yml logs -f app

# 중지
docker-compose -f docker-compose-optimized.yml down
```

## 🔧 주요 변경 사항

1. **web-server 제거**: Streamlit이 포트 80으로 직접 서빙
2. **cloudflared 제거**: 선택사항이므로 제거
3. **dashboard + ai-core 통합**: 하나의 컨테이너에서 두 프로세스 실행
   - Streamlit: 포트 80에서 대시보드 서빙
   - AI-core: 백그라운드에서 비디오 처리 및 자동 파인튜닝

## ✅ 기능 유지

모든 기능이 그대로 유지됩니다:

- ✅ 대시보드 (Streamlit)
- ✅ 비디오 처리 (AI-core)
- ✅ 자동 파인튜닝 (APScheduler)
- ✅ 데이터베이스 (PostgreSQL)
- ✅ Kakao Map API
- ✅ Phi-3-mini-int4 챗봇
- ✅ 인증 시스템
- ✅ 관리자 기능

## ⚠️ 주의사항

1. **포트 80**: Docker 컨테이너는 root로 실행되므로 포트 80 사용 가능
2. **프로세스 관리**: start.sh가 두 프로세스를 관리하고 자동 재시작
3. **리소스**: 하나의 컨테이너에서 두 프로세스 실행하므로 리소스 모니터링 권장

## 🔄 롤백

기존 5컨테이너 구조로 되돌리려면:

```powershell
docker-compose -f docker-compose-optimized.yml down
docker-compose up -d
```
