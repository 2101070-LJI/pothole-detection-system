# 과적합 방지 전략

## 문제점

포트홀이 감지된 이미지만으로 파인튜닝을 하면 모델이 특정 이미지에 과적합(overfitting)될 수 있습니다.

## 해결 방안

### 1. 데이터 증강 (Data Augmentation) ✅

YOLOv8의 내장 데이터 증강 기능을 활용하여 데이터 다양성 증가:

- **HSV 색상 증강**: 색상, 채도, 밝기 변화
- **기하학적 변환**: 회전, 이동, 크기 조정, 전단 변환
- **좌우 뒤집기**: 50% 확률로 좌우 반전
- **모자이크 증강**: 50% 확률로 모자이크 적용
- **MixUp 증강**: 10% 확률로 이미지 혼합

**설정값**:
```python
hsv_h=0.015,      # 색상 변화
hsv_s=0.7,        # 채도 변화
hsv_v=0.4,        # 밝기 변화
degrees=10,       # 회전 각도 (±10도)
translate=0.1,    # 이동 (10%)
scale=0.5,        # 크기 조정
shear=2,          # 전단 변환
fliplr=0.5,       # 좌우 뒤집기 (50%)
mosaic=0.5,       # 모자이크 (50%)
mixup=0.1         # MixUp (10%)
```

### 2. Train/Validation 분리 ✅

데이터셋을 학습용(80%)과 검증용(20%)으로 분리:

- **Train 세트**: 모델 학습에 사용
- **Validation 세트**: 학습 중 성능 평가 및 Early stopping에 사용
- **분리 비율**: 80:20 (포지티브 샘플과 배경 샘플 각각 분리)

### 3. Early Stopping ✅

검증 손실이 개선되지 않으면 학습 조기 종료:

- **Patience**: 15 에포크 (기본값 10에서 증가)
- 검증 손실이 15 에포크 동안 개선되지 않으면 학습 중단
- 최적 모델 자동 저장

### 4. 학습률 조정 ✅

낮은 학습률로 안정적인 학습:

- **초기 학습률 (lr0)**: 0.001 (기본값보다 낮음)
- **최종 학습률 비율 (lrf)**: 0.01
- **워밍업**: 3 에포크 동안 점진적 학습률 증가

### 5. 정규화 (Regularization) ✅

- **가중치 감쇠 (Weight Decay)**: 0.0005 (L2 정규화)
- **모멘텀**: 0.937

### 6. 배경 이미지 (네거티브 샘플) ✅

승인되지 않은 이미지를 배경 이미지로 사용:

- 포지티브 샘플: 승인된 포트홀 이미지 (라벨 있음)
- 네거티브 샘플: 승인되지 않은 이미지 (빈 라벨)
- 비율: 포지티브 샘플의 약 50% (최대 500개)

### 7. 에포크 수 조정 ✅

- **최대 에포크**: 100 (기본값 50에서 증가)
- Early stopping으로 실제로는 더 적게 학습될 수 있음
- 충분한 학습 기회 제공

## 효과

### 과적합 방지

1. **데이터 다양성 증가**: 증강으로 인해 더 다양한 이미지로 학습
2. **일반화 성능 향상**: Validation 세트로 모델 성능 평가
3. **조기 종료**: 과적합 시작 전 학습 중단
4. **정규화**: 가중치 감쇠로 모델 복잡도 제어

### 예상 결과

- **Train Loss**: 점진적으로 감소
- **Val Loss**: 초기 감소 후 안정화 또는 증가 시 Early stopping
- **mAP (mean Average Precision)**: 검증 세트에서 측정
- **일반화 성능**: 새로운 이미지에서도 좋은 성능

## 모니터링

파인튜닝 중 다음을 확인하세요:

1. **학습 곡선**: Train/Val Loss 비교
2. **Early Stopping**: 몇 에포크에서 종료되었는지
3. **mAP 점수**: 검증 세트에서의 성능
4. **과적합 징후**: Val Loss가 증가하기 시작하면 조기 종료

## 추가 개선 방안

### 단기 개선

1. **더 많은 배경 이미지 수집**
   - 승인되지 않은 이미지 외에도 일반 도로 이미지 추가
   - 다양한 환경 조건의 이미지 (날씨, 시간대, 계절)

2. **데이터 균형 조정**
   - 포지티브:네거티브 비율 조정 (예: 1:2 또는 1:3)

3. **교차 검증 (Cross-Validation)**
   - K-fold 교차 검증으로 더 안정적인 평가

### 장기 개선

1. **전이 학습 (Transfer Learning)**
   - 사전 학습된 모델의 가중치 활용
   - 일부 레이어만 파인튜닝 (Freeze)

2. **앙상블 학습**
   - 여러 모델의 예측 결합

3. **하이퍼파라미터 튜닝**
   - 학습률, 배치 크기, 증강 강도 등 최적화

## 현재 설정 요약

```python
# 파인튜닝 설정
epochs=100              # 최대 에포크

weight_decay=0.0005    # 가중치 감쇠
split=0.2              # Train/Val 분리 (20% 검증)

# 데이터 증강
hsv_h=0.015, hsv_s=0.7, hsv_v=0.4  # 색상 증강
degrees=10, translate=0.1, scale=0.5  # 기하학적 변환
fliplr=0.5, mosaic=0.5, mixup=0.1  # 추가 증강
```

이 설정으로 과적합을 효과적으로 방지할 수 있습니다.
