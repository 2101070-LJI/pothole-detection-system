# 포트홀 합성 이미지 자동 생성 실행 방법

## 🚀 빠른 시작

### 방법 1: PowerShell 스크립트 사용 (가장 간단)

```powershell
# 새로운 합성 데이터셋 생성
.\generate_synthetic_potholes.ps1

# 또는 파라미터 지정
.\generate_synthetic_potholes.ps1 -NumPotholes 2 -MinScale 0.3 -MaxScale 2.0
```

### 방법 2: Docker 명령어 직접 실행

```powershell
# 기본 실행 (이미지당 포트홀 1개)
docker exec deep-guardian-ai python /app/generate_synthetic_potholes.py --road_images /app/shared_images/detections --output_dir /app/models/synthetic_dataset --num_potholes 1

# 여러 개의 포트홀, 다양한 크기
docker exec deep-guardian-ai python /app/generate_synthetic_potholes.py --road_images /app/shared_images/detections --output_dir /app/models/synthetic_dataset --num_potholes 2 --min_scale 0.3 --max_scale 2.0
```

### 방법 3: 기존 파인튜닝 데이터셋에 추가

```powershell
# PowerShell 스크립트 사용
.\add_synthetic_to_dataset.ps1

# 또는 Docker 명령어
docker exec deep-guardian-ai python /app/add_synthetic_potholes.py --road_images /app/shared_images/detections --dataset_dir /app/models/finetune_dataset --num_potholes 1
```

## 📋 파라미터 설명

### 기본 파라미터

- `--road_images`: 포트홀이 없는 도로 이미지가 있는 디렉토리
  - 기본값: `/app/shared_images/detections`
  
- `--output_dir`: 합성된 이미지와 라벨이 저장될 디렉토리
  - 기본값: `/app/models/synthetic_dataset`

- `--num_potholes`: 각 이미지에 합성할 포트홀 개수
  - 기본값: `1`
  - 권장: 1-3개

- `--min_scale`: 포트홀 최소 크기 (배율)
  - 기본값: `0.5`
  - 권장: 0.3-0.8

- `--max_scale`: 포트홀 최대 크기 (배율)
  - 기본값: `1.5`
  - 권장: 1.0-2.0

## 💡 사용 예시

### 예시 1: 기본 실행
```powershell
.\generate_synthetic_potholes.ps1
```
- 도로 이미지당 포트홀 1개 생성
- 크기: 0.5~1.5배

### 예시 2: 다양한 크기의 포트홀
```powershell
.\generate_synthetic_potholes.ps1 -NumPotholes 2 -MinScale 0.3 -MaxScale 2.0
```
- 도로 이미지당 포트홀 2개 생성
- 크기: 0.3~2.0배 (더 다양한 크기)

### 예시 3: 대량 생성
```powershell
# PowerShell에서 Docker 명령어 직접 실행
docker exec deep-guardian-ai python /app/generate_synthetic_potholes.py `
  --road_images /app/shared_images/detections `
  --output_dir /app/models/synthetic_dataset `
  --num_potholes 1 `
  --min_scale 0.4 `
  --max_scale 1.8
```

## 📂 출력 결과

실행 후 다음 디렉토리에 결과가 저장됩니다:

```
/app/models/synthetic_dataset/
├── images/
│   └── train/
│       ├── image1.jpg  (포트홀이 합성된 이미지)
│       ├── image2.jpg
│       └── ...
└── labels/
    └── train/
        ├── image1.txt  (YOLO 형식 라벨)
        ├── image2.txt
        └── ...
```

각 `.txt` 파일 형식:
```
0 0.523456 0.345678 0.123456 0.098765
```
- `0`: 클래스 ID (포트홀)
- 다음 4개 숫자: 중심 X, 중심 Y, 너비, 높이 (정규화된 좌표, 0-1 범위)

## ⚠️ 주의사항

1. **컨테이너 실행 확인**
   ```powershell
   docker ps --filter "name=deep-guardian-ai"
   ```
   컨테이너가 실행 중이어야 합니다.

2. **도로 이미지 준비**
   - `shared_images/detections` 디렉토리에 포트홀이 없는 도로 이미지가 있어야 합니다.
   - 지원 형식: `.jpg`, `.jpeg`, `.png`

3. **디스크 공간**
   - 합성된 이미지가 저장될 충분한 공간이 필요합니다.
   - 이미지 1개당 약 100KB-500KB

## 🔧 문제 해결

### 컨테이너가 실행되지 않는 경우
```powershell
docker-compose up -d ai-core
```

### 권한 오류가 발생하는 경우
```powershell
# Windows에서 디렉토리 권한 확인
docker exec deep-guardian-ai ls -la /app/shared_images/detections
```

### 이미지가 생성되지 않는 경우
```powershell
# 로그 확인
docker-compose logs ai-core --tail 50
```
