# 컨테이너 통합 가이드

## 🎯 통합 목표

**5개 컨테이너 → 2개 컨테이너로 축소**

### 통합 전 (5개)
1. web-server (Apache)
2. dashboard (Streamlit)
3. ai-core (AI Engine)
4. db (PostgreSQL)
5. cloudflared (Cloudflare Tunnel)

### 통합 후 (2개)
1. **app** (Dashboard + AI-core 통합)
2. **db** (PostgreSQL)

## ✅ 통합 내용

### 1. web-server 제거
- **이유**: Streamlit이 자체 웹 서버를 제공하므로 Apache 프록시 불필요
- **변경**: Streamlit을 포트 80으로 직접 서빙

### 2. cloudflared 제거
- **이유**: 외부 접근용 선택사항이므로 제거
- **참고**: 필요시 별도로 실행 가능

### 3. dashboard + ai-core 통합
- **방법**: 하나의 컨테이너에서 두 프로세스를 동시 실행
  - Streamlit: 포트 80에서 대시보드 서빙
  - AI-core: 백그라운드에서 비디오 처리 및 자동 파인튜닝

## 📁 새로운 구조

```
app/
├── Dockerfile          # 통합 컨테이너 빌드 파일
├── requirements.txt    # 통합 의존성 (dashboard + ai-core)
├── start.sh           # 시작 스크립트 (두 프로세스 실행)
└── .dockerignore      # 빌드 제외 파일
```

## 🚀 사용 방법

### 1. 최적화된 docker-compose 사용

```powershell
# 최적화된 버전으로 실행
docker-compose -f docker-compose-optimized.yml up -d

# 빌드 포함
docker-compose -f docker-compose-optimized.yml up -d --build
```

### 2. 기존 버전과 비교

```powershell
# 기존 5컨테이너 버전
docker-compose up -d

# 최적화된 2컨테이너 버전
docker-compose -f docker-compose-optimized.yml up -d
```

## 🔧 주요 변경 사항

### Dockerfile
- CUDA 기반 이미지 사용 (AI-core 요구사항)
- dashboard와 ai-core 의존성 통합
- 시작 스크립트로 두 프로세스 동시 실행

### start.sh
- AI-core를 백그라운드로 실행
- Streamlit을 포트 80으로 실행
- 프로세스 모니터링 및 자동 재시작

### docker-compose-optimized.yml
- `app` 서비스: dashboard + ai-core 통합
- `db` 서비스: PostgreSQL (변경 없음)
- web-server, cloudflared 제거

## ⚠️ 주의사항

### 1. 포트 80 권한
- Docker 컨테이너는 기본적으로 root로 실행되므로 포트 80 사용 가능
- 보안이 중요한 경우 비root 사용자로 변경 고려

### 2. 프로세스 관리
- start.sh가 두 프로세스를 관리
- 프로세스가 종료되면 자동으로 재시작

### 3. 리소스 사용
- 하나의 컨테이너에서 두 프로세스 실행
- 메모리 및 CPU 사용량 모니터링 권장

## 🔄 롤백 방법

기존 5컨테이너 구조로 되돌리려면:

```powershell
# 최적화 버전 중지
docker-compose -f docker-compose-optimized.yml down

# 기존 버전 실행
docker-compose up -d
```
