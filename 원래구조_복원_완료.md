# 원래 구조 복원 완료

## ✅ 복원된 구조

원래 5개 컨테이너 구조로 복원되었습니다:

1. **web-server** (Apache Reverse Proxy) - 포트 80
2. **dashboard** (Streamlit) - 포트 8501
3. **ai-core** (AI Engine) - YOLOv8 + NPU Worker 호출
4. **db** (PostgreSQL) - 포트 5432
5. **cloudflared** (Cloudflare Tunnel) - 외부 접근

## 📝 변경 사항

### 1. docker-compose.yml
- 2개 컨테이너 구조 → 5개 컨테이너 구조로 복원
- PostgreSQL 데이터베이스 사용
- Apache 웹서버 추가
- Streamlit 대시보드 분리
- Cloudflare Tunnel 추가

### 2. 데이터베이스 설정
- `django_app/settings.py`: 기본값을 PostgreSQL로 변경
- `ai-core/main.py`: 기본 DATABASE_URL을 PostgreSQL로 변경

### 3. 데이터베이스 초기화
- `database/init.sql` 사용 (PostgreSQL용)

## 🚀 실행 방법

### 1. 전체 시스템 시작

```powershell
docker-compose up -d --build
```

### 2. 개별 서비스 확인

```powershell
# 로그 확인
docker-compose logs -f dashboard
docker-compose logs -f ai-core
docker-compose logs -f web-server

# 상태 확인
docker-compose ps
```

### 3. 웹 접속

- **Apache (Reverse Proxy)**: http://localhost
- **Streamlit 대시보드 (직접 접속)**: http://localhost:8501
- **PostgreSQL**: localhost:5432

## 📋 서비스 설명

### 1. web-server (Apache)
- **포트**: 80
- **역할**: Reverse Proxy, 외부 진입점
- **설정**: `apache/httpd-vhost.conf`
- **프록시 대상**: dashboard:8501

### 2. dashboard (Streamlit)
- **포트**: 8501
- **역할**: 포트홀 데이터 시각화 및 관리
- **기능**: 지도 시각화, 통계 대시보드, 데이터 테이블

### 3. ai-core (AI Engine)
- **역할**: 포트홀 탐지 및 검증
- **처리 흐름**:
  1. YOLOv8로 포트홀 탐지
  2. 포트홀 영역 크롭
  3. NPU Worker 호출 (깊이 검증)
  4. 검증 통과 데이터만 DB 저장
- **환경 변수**:
  - `NPU_WORKER_URL`: http://host.docker.internal:9001/depth
  - `DATABASE_URL`: postgresql://pothole_user:pothole_pass@db:5432/pothole_db
  - `KAKAO_MAP_APP_KEY`: 카카오 맵 API 키

### 4. db (PostgreSQL)
- **포트**: 5432
- **데이터베이스**: pothole_db
- **사용자**: pothole_user / pothole_pass
- **초기화**: `database/init.sql`

### 5. cloudflared (Cloudflare Tunnel)
- **역할**: 외부 접근을 위한 터널
- **설정**: `cloudflared/config.yml`
- **프록시 대상**: web-server:80

## ⚠️ 주의사항

1. **NPU Worker**: Windows Host에서 별도로 실행해야 합니다
   ```powershell
   .\start_npu_worker.ps1
   ```

2. **Cloudflare Tunnel**: `cloudflared/credentials.json`과 `cert.pem` 파일이 필요합니다

3. **데이터베이스**: PostgreSQL을 사용하므로 `psycopg2-binary` 패키지가 필요합니다
