## 🎯 주요 기능 확인

### ✅ 완전히 동작하는 기능

1. **포트홀 탐지 (AI Core)**
   - YOLOv8 모델을 사용한 실시간 포트홀 탐지
   - 이미지/비디오 처리 파이프라인
   - GPS 좌표 추출 (EXIF 데이터)

2. **깊이 검증 (NPU Worker)**
   - OpenVINO 기반 Depth Anything V2 모델
   - NPU 우선 사용 (AUTO:NPU,GPU,CPU)
   - 깊이 비율 계산 및 검증

3. **위험도 평가**
   - Kakao Map API 연동
   - 좌표 → 주소 변환
   - 주변 시설물 검색 (학교, 병원)
   - 위험도 레벨 결정 (critical, high, medium, low)

4. **데이터 시각화 (Dashboard)**
   - 실시간 통계 카드
   - Folium 기반 인터랙티브 지도
   - 데이터 필터링 및 테이블
   - 차트 시각화
   - 이미지 뷰어

5. **데이터 저장 (PostgreSQL)**
   - 포트홀 데이터 저장
   - 위험도 및 우선순위 정보
   - 위치 정보 및 메타데이터

## 🔧 시스템 사용 방법

### 1. 대시보드 접근

브라우저에서 다음 주소로 접근:
```
http://localhost:8501
```

### 2. NPU Worker 확인

```powershell
# Health Check
Invoke-RestMethod http://localhost:9001/health

# 예상 응답:
# {
#   "status": "healthy",
#   "model_loaded": true,
#   "device": "AUTO:NPU,GPU,CPU"
# }
```

### 3. 컨테이너 상태 확인

```powershell
# 전체 컨테이너 상태
docker-compose ps

# 개별 로그 확인
docker-compose logs -f ai-core
docker-compose logs -f dashboard
docker-compose logs -f db
```

### 4. 데이터베이스 접속

```powershell
# PostgreSQL 접속
docker-compose exec db psql -U pothole_user -d pothole_db

# 데이터 확인
docker-compose exec db psql -U pothole_user -d pothole_db -c "SELECT COUNT(*) FROM potholes;"
```

## ⚠️ 주의 사항

### 1. Nginx Health Check

- Nginx 컨테이너가 "unhealthy"로 표시될 수 있음
- 하지만 대시보드는 직접 접근 가능하므로 기능상 문제 없음
- http://localhost:8501로 접근하면 정상 동작

### 2. NPU Worker 종료

- NPU Worker는 별도 PowerShell 터미널에서 실행 중
- 종료하려면 해당 터미널에서 `Ctrl+C` 입력
- 종료 시 깊이 검증 기능이 동작하지 않음

### 3. Docker 컨테이너 재시작

```powershell
# 전체 재시작
docker-compose restart

# 개별 재시작
docker-compose restart ai-core
docker-compose restart dashboard
```
