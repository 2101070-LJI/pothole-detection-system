# Deep-Guardian 2-Container 구조 가이드

## 📋 개요

기존 5개 컨테이너 구조에서 **2개 컨테이너 구조**로 변경되었습니다:

1. **inference-container**: AI 모델 추론 전용 컨테이너
2. **lamp-container**: LAMP 스택 (Linux, Apache, MySQL, Python)

---

## 🏗️ 새로운 아키텍처

```
[사용자 브라우저]
    ↓
[LAMP Container]
├── Apache (포트 80) - 웹서버
├── MySQL (포트 3306) - 데이터베이스
└── Python/Flask - 웹 애플리케이션
    ↓
[Inference Container]
├── YOLOv8 - 포트홀 탐지
└── NPU Worker 호출 - 깊이 검증
```

## 🔧 컨테이너 상세

### 1. Inference Container

**역할**: AI 모델 추론 전용

**기능**:
- YOLOv8 포트홀 탐지
- NPU Worker 호출 (깊이 검증)
- 데이터베이스 저장 (MySQL)

**환경 변수**:
- `NPU_WORKER_URL`: NPU Worker URL
- `YOLO_MODEL_PATH`: YOLOv8 모델 경로
- `DATABASE_URL`: MySQL 연결 URL

**볼륨**:
- `./ai-core`: AI Core 코드
- `./ai-core/models`: 모델 파일
- `./shared_images`: 공유 이미지
- `./ai-core/videos`: 입력 비디오

### 2. LAMP Container

**역할**: 웹서버, 데이터베이스, 웹 애플리케이션

**구성**:
- **Linux**: Ubuntu 22.04
- **Apache**: 웹서버 (포트 80)
- **MySQL**: 데이터베이스 (포트 3306)
- **Python**: Flask 웹 애플리케이션

**기능**:
- 웹 대시보드 제공
- REST API 제공 (`/api/potholes`)
- 정적 파일 서빙
- 데이터베이스 관리

**환경 변수**:
- `MYSQL_ROOT_PASSWORD`: MySQL root 비밀번호
- `MYSQL_DATABASE`: 데이터베이스 이름
- `MYSQL_USER`: MySQL 사용자
- `MYSQL_PASSWORD`: MySQL 비밀번호
- `GEMINI_API_KEY`: Gemini API 키
- `OLLAMA_URL`: Ollama URL

## 🚀 실행 방법

### 1. 컨테이너 빌드 및 시작

```powershell
# Docker Compose로 빌드 및 시작
docker-compose up -d --build
```

### 2. 컨테이너 상태 확인

```powershell
# 컨테이너 상태 확인
docker-compose ps

# 로그 확인
docker-compose logs -f
```

### 3. 웹 접속

- **웹 대시보드**: http://localhost
- **API 엔드포인트**: http://localhost/api/potholes
- **헬스 체크**: http://localhost/health

## 🔄 데이터베이스 마이그레이션

### PostgreSQL → MySQL 변경

기존 PostgreSQL 데이터베이스에서 MySQL로 마이그레이션:

1. **데이터 내보내기** (PostgreSQL):
```sql
COPY potholes TO '/tmp/potholes.csv' CSV HEADER;
```

2. **데이터 가져오기** (MySQL):
```sql
LOAD DATA INFILE '/tmp/potholes.csv'
INTO TABLE potholes
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS;
```

### Django 설정 변경

`django_app/settings.py`가 자동으로 MySQL을 지원하도록 수정되었습니다:
- `DATABASE_URL`이 `mysql://`로 시작하면 MySQL 사용
- `postgresql://`로 시작하면 PostgreSQL 사용

## 🔧 설정 파일

### docker-compose.yml

주요 변경사항:
- 5개 서비스 → 2개 서비스
- PostgreSQL → MySQL
- Streamlit → Flask 웹 애플리케이션
- Apache와 MySQL이 같은 컨테이너에 통합

### 환경 변수 (.env)

```env
GEMINI_API_KEY=your_key_here
GEMINI_MODEL=gemini-2.0-flash
OLLAMA_URL=http://host.docker.internal:11434
OLLAMA_MODEL=llama3.2
```

## 🐛 문제 해결

### MySQL 연결 오류

1. **컨테이너 재시작**:
```powershell
docker-compose restart lamp
```

2. **MySQL 로그 확인**:
```powershell
docker-compose logs lamp | grep mysql
```

3. **수동 연결 테스트**:
```powershell
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db
```

### Apache 오류

1. **Apache 설정 확인**:
```powershell
docker exec -it deep-guardian-lamp apache2ctl configtest
```

2. **Apache 재시작**:
```powershell
docker exec -it deep-guardian-lamp service apache2 restart
```

### Flask 애플리케이션 오류

1. **Python 패키지 확인**:
```powershell
docker exec -it deep-guardian-lamp pip list
```

2. **애플리케이션 로그 확인**:
```powershell
docker-compose logs lamp | grep app
```

## 📝 주의사항

1. **데이터 백업**: 기존 PostgreSQL 데이터를 백업하세요
2. **환경 변수**: `.env` 파일의 데이터베이스 URL이 MySQL을 가리키는지 확인
3. **포트 충돌**: 기존 PostgreSQL이 실행 중이면 포트 5432 충돌 가능
4. **볼륨 마운트**: `mysql_data` 볼륨이 생성되어 데이터가 영구 저장됨
