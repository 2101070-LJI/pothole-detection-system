# Deep-Guardian 프로젝트 실행 명령어

## 🚀 빠른 시작

### 1. 전체 시스템 시작 (권장)

```powershell
# 프로젝트 디렉토리로 이동
cd C:\Users\your_username\Desktop\test

# Docker Compose로 빌드 및 시작
docker-compose up -d --build
```

### 2. 기본 사용자 생성

```powershell
# 컨테이너 내부에서 기본 사용자 생성
docker exec -it deep-guardian-lamp python3 /var/www/app/create_default_users.py
```

### 3. 웹 접속

브라우저에서 **http://localhost** 접속

**기본 계정**:
- 관리자: `admin` / `admin123`
- 사용자: `user` / `user123`

## 📋 주요 명령어

### 컨테이너 관리

```powershell
# 전체 시스템 시작
docker-compose up -d

# 전체 시스템 중지
docker-compose down

# 전체 시스템 재시작
docker-compose restart

# 특정 컨테이너만 재시작
docker-compose restart lamp
docker-compose restart inference

# 컨테이너 상태 확인
docker-compose ps

# 컨테이너 로그 확인
docker-compose logs -f

# 특정 컨테이너 로그만 확인
docker-compose logs -f lamp
docker-compose logs -f inference
```

### 컨테이너 내부 접속

```powershell
# LAMP 컨테이너 접속
docker exec -it deep-guardian-lamp bash

# Inference 컨테이너 접속
docker exec -it deep-guardian-inference bash
```

### 데이터베이스 관리

```powershell
# MySQL 접속
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db

# 데이터베이스 목록 확인
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass -e "SHOW DATABASES;"

# 테이블 목록 확인
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db -e "SHOW TABLES;"

# 포트홀 데이터 개수 확인
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db -e "SELECT COUNT(*) FROM potholes;"
```

### 사용자 관리

```powershell
# 기본 사용자 생성/업데이트
docker exec -it deep-guardian-lamp python3 /var/www/app/create_default_users.py

# 사용자 목록 확인 (MySQL)
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db -e "SELECT id, username, role FROM users;"
```

### 볼륨 관리

```powershell
# 볼륨 목록 확인
docker volume ls

# 볼륨 상세 정보
docker volume inspect test_mysql_data

# 볼륨 삭제 (주의: 데이터 삭제됨)
docker volume rm test_mysql_data
```

### 이미지 관리

```powershell
# 이미지 목록
docker images

# 이미지 재빌드 (캐시 없이)
docker-compose build --no-cache

# 특정 서비스만 재빌드
docker-compose build --no-cache lamp
docker-compose build --no-cache inference
```

## 🔧 문제 해결 명령어

### 포트 충돌 확인

```powershell
# 포트 80 확인
netstat -ano | findstr :80

# 포트 3306 확인
netstat -ano | findstr :3306

# 포트 9001 확인 (NPU Worker)
netstat -ano | findstr :9001
```

### 컨테이너 상태 확인

```powershell
# 실행 중인 컨테이너 확인
docker ps

# 모든 컨테이너 확인 (중지된 것 포함)
docker ps -a

# 컨테이너 리소스 사용량
docker stats
```

### 로그 확인

```powershell
# 최근 100줄 로그
docker-compose logs --tail=100

# 실시간 로그 (모든 컨테이너)
docker-compose logs -f

# 특정 컨테이너 로그
docker logs deep-guardian-lamp
docker logs deep-guardian-inference

# 에러만 필터링
docker-compose logs | findstr -i error
```

### 컨테이너 재시작

```powershell
# 전체 재시작
docker-compose restart

# 특정 컨테이너 재시작
docker restart deep-guardian-lamp
docker restart deep-guardian-inference

# 컨테이너 중지 후 시작
docker-compose down
docker-compose up -d
```

### 데이터베이스 초기화

```powershell
# MySQL 재시작
docker exec -it deep-guardian-lamp service mysql restart

# 데이터베이스 재생성 (주의: 모든 데이터 삭제)
docker-compose down -v
docker-compose up -d
```

## 🧪 테스트 명령어

### API 테스트

```powershell
# 헬스 체크
curl http://localhost/health

# 인증 상태 확인
curl http://localhost/api/auth/status

# 포트홀 데이터 조회
curl http://localhost/api/potholes

# 통계 조회
curl http://localhost/api/statistics?days=7

# 챗봇 테스트
curl -X POST http://localhost/api/chat -H "Content-Type: application/json" -d "{\"message\": \"오늘 탐지된 포트홀은 몇 개인가요?\"}"
```

### PowerShell에서 API 테스트

```powershell
# 헬스 체크
Invoke-RestMethod -Uri http://localhost/health

# 포트홀 데이터 조회
Invoke-RestMethod -Uri http://localhost/api/potholes

# 로그인
$body = @{username='admin'; password='admin123'} | ConvertTo-Json
Invoke-RestMethod -Uri http://localhost/api/auth/login -Method POST -Body $body -ContentType 'application/json'
```

## 🔄 개발 워크플로우

### 코드 수정 후 재시작

```powershell
# 1. 코드 수정 후
# 2. 컨테이너 재시작 (볼륨 마운트되어 있어서 자동 반영)
docker-compose restart lamp

# 또는 전체 재시작
docker-compose restart
```

### 새 패키지 설치

```powershell
# LAMP 컨테이너에 패키지 설치
docker exec -it deep-guardian-lamp pip3 install 패키지명

# requirements.txt 업데이트 후 재빌드
docker-compose build lamp
docker-compose up -d lamp
```

### 환경 변수 변경

```powershell
# .env 파일 수정 후
docker-compose down
docker-compose up -d
```

## 🛠️ 유지보수 명령어

### 정리 작업

```powershell
# 사용하지 않는 이미지 삭제
docker image prune

# 사용하지 않는 볼륨 삭제
docker volume prune

# 전체 정리 (주의: 사용하지 않는 모든 것 삭제)
docker system prune -a
```

### 백업

```powershell
# MySQL 데이터베이스 백업
docker exec deep-guardian-lamp mysqldump -u pothole_user -ppothole_pass pothole_db > backup_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql

# 볼륨 백업
docker run --rm -v test_mysql_data:/data -v ${PWD}:/backup alpine tar czf /backup/mysql_backup.tar.gz /data
```

### 복원

```powershell
# MySQL 데이터베이스 복원
docker exec -i deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db < backup.sql
```

## 🚨 긴급 상황 대응

### 컨테이너가 시작되지 않을 때

```powershell
# 1. 로그 확인
docker-compose logs

# 2. 컨테이너 상태 확인
docker-compose ps

# 3. 강제 재시작
docker-compose down
docker-compose up -d --force-recreate

# 4. 캐시 없이 재빌드
docker-compose build --no-cache
docker-compose up -d
```

### 데이터베이스 연결 오류

```powershell
# MySQL 재시작
docker exec -it deep-guardian-lamp service mysql restart

# 연결 테스트
docker exec -it deep-guardian-lamp mysql -u pothole_user -ppothole_pass pothole_db -e "SELECT 1;"
```

### 웹 페이지가 열리지 않을 때

```powershell
# Apache 상태 확인
docker exec -it deep-guardian-lamp service apache2 status

# Apache 재시작
docker exec -it deep-guardian-lamp service apache2 restart

# Apache 설정 테스트
docker exec -it deep-guardian-lamp apache2ctl configtest
```

## ⚠️ 주의사항

1. **포트 충돌**: 기존 서비스와 포트가 충돌하지 않는지 확인
2. **볼륨 삭제**: `docker-compose down -v`는 모든 데이터를 삭제합니다
3. **비밀번호**: 프로덕션 환경에서는 반드시 기본 비밀번호를 변경하세요
4. **백업**: 중요한 데이터는 정기적으로 백업하세요
